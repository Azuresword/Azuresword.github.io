title: 基于TDMA技术和UWB的多机器人实时定位系统

基于 TDMA 技术和 UWB技术的多机器人协同控制系统

课题研究背景及其意义

室内定位及导航技术

室内定位技术1是一种基于位置的服务（location based service）LSB 技术
在现代社会己经有了广泛的应用，人们的生活及生产大多数开始随着社会的发展 发生在室内，那么室内的LSB技术作为非常必要的应用场景值得深入研究，目前在商场定位 2， 停车场寻车3，等方方面面都有着应用和落地场景。

现有室内定位导航方案有

目前室内导航与定位技术 已有的方案 基于 RFID WLAN 等非测距的定位方式，以及主动式的SLAM定位方法。但是这些技术 都有着不同的利弊 成本较高或是稳定性精度不足 。

高精度室内定位和导航技术的研究

UWB技术是近年兴起的超带宽通信技术可以实现高精度的定位需求。一方面具有穿透能力高的特点，一方面能够满足使用TDMA技术时能够完成多目标的检测，且具有高精度的特点，

频谱复用技术当前已经有很多种理论方案 相比CDMA以及FDMA等 TDMA方案具有可拓展性高，

本文中我们设计的技术是基于TDMA技术和UWB技术以及独立机器人控制系统实现的多机器人协作系统，他的核心同样是基于室内定位和运动系统，在方案得到完整验证后在实际生产生活中也有着广泛的使用价值。

多机器人协作控制系统概述

多机器人系统控制系统简介

多机器人协作系统是近些年兴起的技术领域，多机器人协作的方式目前在大类上被划分为 ：主动控制式和分布式控制式，分布式控制即每台机器都能够单独在系统中完成任务的。在我们的控制场域中单台机器首先实现了P2P move任务，多台机在TDMA算法的支持下能够完成同时在同一场域内的运转，最终再结合上位机发布和多机互信机制能够实现多机器人的系统控制。

TDMA技术简介

TDMA时分多址（Time Division Multiple Access）

在GSM移动通信中采用较多 ，时分多址是把时间分割成周期性的帧(Frame)每一个帧再分割成若干个时隙向基站发送信号，在满足定时和同步的条件下，基站可以分别在各时隙中接收到各移动终端的信号而不混扰。

在本文描述的系统中TDMA技术的使用使得系统能够在时间片轮转算法的支持下每一个UWB标签都能够按照既定需求访问基站从而获得距离信息。

UWB技术简介

超宽带（Ultra Wide-Band，UWB）是一种新型的无线通信技术，根据美国联邦通信委员会的规范，UWB的工作频带为3.1~10.6GHz，系统-10dB带宽与系统中心频率之比大于20%或系统带宽至少为500MHz。超宽带的主要优势有，低功耗、对信道衰落（如多径、非视距等信道）不敏感、抗干扰能力强、不会对同一环境下的其他设备产生干扰、穿透性较强（能在穿透一堵砖墙的环境进行定位），具有很高的定位准确度和定位精度。

在我们的系统中采用 UWB-TDOA三维定位模式，最终精度能达到5m场域内

系统架构

整体系统设计

​ 本文提出的方案以Mobile Robot 为核心的控制系统 。自身位置的确定由UWB基站和UWB标签的TDOA算法测定。上位机

​ Mobile Robot 将数据接收器收到的数据转换为期望值并将数据压入消息队列传送给执行器，执行器依据期望值和目标值进行对比得出最终驱动转速，使电机能够按照预期转动进行闭环控制，从而实现单机的P2P运动。

​ 同时Mobile Robot具备Esp8266无线通信模组，NRF24L01无线通信模组，数传发送机。

​ 在外部监测和TDMA算法的实现上采用外部上位机的设计，外部上位机装配有NRF24L01，无线数传，ESP8266模组。

​ 在上位机中在需要对单机快速检测时进行点对点数传对频后可读取数传中回传的机器人信息，上位机中解算出TDMA算法所需要的时间差分信息后，通过NRF24L01模组进行信息广播，信息广播后Mobile Robot即可获取到此刻是否应当使能UWB标签获取信息，如果此时正在通信则返回占用信息，具体的收发数据情况和数据帧如下。

Frame control Time stream Target Request
2 4 4 2

Frame control 为数据帧次数 ，由TDMA算法节点决定，TDMA算法提供时间片轮转的策略。

Time stream 为单次数据收发的时间，由Mobile Robot反馈测算得到的通信延迟进行测定。

Target为发送对象，由于NRF24L01是广播通讯机制，所以全部机器人都会由此帧去确认通信对象。

Request为确认通信方向，由下位机向上位机Request为0，上位机向下位机为1.

在矩形场地的四个顶点设置 UWB基站 ，每台Mobile Robot 包含一个UWB标签 UWB基站之间完成位置信息测算，UWB基站核心射频模块为DW1000芯片，在STM32F1芯片及其外部电路中进行处理后得出自身位置通过串口下发至Mobile Robot的Data transfer程序执行器，在Mobile Robot 的处理程序中将该部分数据与机器人执行器的闭环控制器相连接，即可实现机器人的基本运动流程。
